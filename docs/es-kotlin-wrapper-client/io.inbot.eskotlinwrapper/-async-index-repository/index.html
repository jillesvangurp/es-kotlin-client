AsyncIndexRepository

class AsyncIndexRepository<T : Any>(indexName: String, client: RestHighLevelClient, modelReaderAndWriter: ModelReaderAndWriter<T>, refreshAllowed: Boolean, type: String?, indexWriteAlias: String, indexReadAlias: String, fetchSourceContext: FetchSourceContext?, defaultRequestOptions: RequestOptions)
Content copied to clipboard

Repository abstraction that allows you to work with indices asynchronously via co-routines.

You should create a Repository for each index you work with. You need to specify a ModelReaderAndWriter for serialization and deserialization.

See also

RestHighLevelClient.asyncIndexRepository for a convenient way to create a repository.

Parameters

defaultRequestOptions

passed on all API calls. Defaults to RequestOptions.DEFAULT. Use this to set custom headers or override on each call on the repository.

fetchSourceContext

if not null, will be passed to Get and Search requests.

indexName

name of the index

indexReadAlias

Alias used for read operations. If you are using aliases, you can separate reads and writes. Defaults to indexName.

indexWriteAlias

Alias used for write operations. If you are using aliases, you can separate reads and writes. Defaults to indexName.

modelReaderAndWriter

serialization of your model class.

refreshAllowed

if false, the refresh will throw an exception. Defaults to false.

T

the type of the object that is stored in the index.

type

the type of the documents in the index; defaults to null. Since ES 6, there can only be one type. Types are deprecated in ES 7 and removed in ES 8.
